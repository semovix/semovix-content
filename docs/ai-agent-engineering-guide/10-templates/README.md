## 写作要求

1. 每篇文章都要先说明“解决什么问题”。
2. 概念解释必须服务于工程落地，避免泛泛而谈。
3. 需要明确对应 Semovix / Xino / AI Workbench 的能力映射。
4. 需要补充可画图的结构，方便后续生成公众号配图、官网图或培训图。
5. 涉及外部项目和技术事实时，应补充到根目录 `REFERENCES.md`。
