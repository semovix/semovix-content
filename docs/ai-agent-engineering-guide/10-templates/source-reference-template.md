# 来源引用模板

## 1. 单个来源记录模板

```markdown
### 来源名称

- 类型：官方文档 / 官方博客 / 论文 / 开源仓库 / 技术文章 / 内部文档
- 地址：
- 作者 / 机构：
- 发布时间：
- 访问日期：
- 适用章节：
- 关键内容：
- 使用方式：直接引用 / 背景参考 / 对比分析 / 图示来源
- 可信度：高 / 中 / 低
```

## 2. 文章末尾参考资料模板

```markdown
## 参考资料

1. OpenAI Agents SDK, Official Documentation.
2. Model Context Protocol Specification, Official Documentation.
3. Anthropic Claude Managed Agents, Official Documentation.
```

## 3. 来源评估标准

| 级别 | 来源类型 | 使用建议 |
|---|---|---|
| 高 | 官方文档、官方规范、论文、开源仓库 README | 可作为关键事实来源 |
| 中 | 官方博客、工程博客、技术白皮书 | 可作为趋势和实践参考 |
| 低 | 个人博客、二手解读、社交媒体 | 只作为补充，不作为关键结论唯一依据 |

## 4. 引用注意事项

1. 不要把趋势判断写成行业定论。
2. 不要引用无法验证的截图或二手转述作为关键依据。
3. 外部技术事实需要记录来源。
4. Semovix 内部判断要标明是“产品规划”还是“已实现能力”。
5. 如果资料可能过期，需要在文中说明版本或访问时间。
